# Карьерная лестница токена
> «Вообще-то у нас в каждом отделе НИИ — свои ожидания к задаванию вопросов, специализациям и багажу знаний. А пройти надо через все отделы».

В каждой избушке свои погремушки.

То есть на каждом слое нейросети всё заново: новые «снаряжающие» матрицы и новый обмен информацией со старыми-новыми знакомыми.

Но порядок быть должон! Токены задают вопросы только предкам, не потомкам.

В итоге перед последним слоем последний токен в цепочке «знает всё» от тех предков, до которых смог дотянуться.

### Удивительное наблюдение
- `Q` токена нужен только самому этому токену…
- …и только пока токен не дойдёт до конца модели

> [!IMPORTANT]
> Наконец-то хоть что-то можно выбросить!
> Да, это только по одному вектору на токен для каждого слоя модели.
> Но всё-таки должно стать полегче.

- А вот `K` и `V` нам ещё много-много раз пригодятся.
- Растём! Кушаем память! Заглядение.

## Цель мероприятия — родить товарища
> «Последний мой слой — это словарь. Если токен пропустить через него, то вы узнаете… Ну, что-нибудь на человеческом».

Модель **научили** так, чтобы «что-нибудь на человеческом» было словом-предсказанием. Словом, идеально подходящим для продолжения цепочки.

Но вообще-то научить могли чему угодно. Придать этой вероятности какой угодно смысл.

Ты мне слово — я те два.

Ожидание: одна женщина, 9 месяцев.
Реальность: все женщины сколько есть, и давайте уже побыстрее!

Модель отвечает связно только потому, что вы скормили ей связный текст. 
Для любителей терминов — ищите термин «маскирование».



## Что там делает `C`?

Ой да давайте просто сходим и посмотрим.

Скомпилируем [динамическую библиотеку](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md).
```bash
git clone git@github.com:ggml-org/llama.cpp.git
cd llama.cpp
cmake -B build -DBUILD_SHARED_LIBS=ON
cmake --build build --config Release -j 4
```

Подключим через [обычный заголовочный файл](https://github.com/ggml-org/llama.cpp/blob/master/include/llama.h) — и дальше пользуемся **как получится**.
