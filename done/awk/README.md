##### Программа — набор правил
##### Правило — условие + действие
…

==условие== { `действие` }

==условие== { `действие` }

==условие== { `действие` }

==условие== { `действие` }

…

##### Правила применяются к записям

`awk` ==<тут параметры и программа>== `файл_с_записями.txt`

##### Запись — набор полей

==$0== == `$1` `$2` `$3` `$4`

##### В умолчаних — и простота, и **засада**
- Шаблон по умолчанию — **каждая** запись
- Действие по умолчанию — вывод в `stdout`
- Запись по умолчанию == **строка**, поле == **слово**

> [!NOTE]
> И дальше всё, как в большом языке.\
> Переменные, ветвления, массивы даже.

###### Запустим хоть что-то по определению
> Давайте пройдём по **строчкам** файла и к **каждой** применим простое **действие** по умолчанию — вывод на экран.
```awk
awk '1' ~/.gitconfig
```

###### Научимся действию
> Выведем для каждой строки её длину в символах.
```awk
awk '{ print length }' ~/.gitconfig
```

###### Увидим поля (и даже колонки)
> Подберёмся к списку параметров — мы знаем, что параметры стоят в начале каждой строки.
```awk
awk '{ print $1 }' ~/.gitconfig
```

> [!CAUTION]
> И это 90% `awk`-кода.\
> Всё, что будет дальше — очень продвинутая тема.

###### Научимся проверять поля по шаблону
> Посмотрим на группы параметров — какие они?
```awk
awk '$1 ~ "\[" {print}' ~/.gitconfig
```

###### Что-нибудь посчитаем (а заодно и используем переменную)
> Сколько у нас групп параметров?
```awk
awk -v sum=0 '$1 ~ "\[" { sum = sum + 1; print sum}' ~/.gitconfig
```
- Вообще-то инициировать переменную — это лишнее, но нельзя травмировать психику.

###### Вынесем отдельно начальные и конечные действия
> Теперь не выводи промежуточные суммы, только финальную.
```awk
awk 'BEGIN {sum = 0} $1 ~ "\[" { sum = sum + 1 } END { print sum}' ~/.gitconfig
```
- В `GNU awk` есть ещё ==BEGINFILE== и ==ENDFILE==

###### Выпускайте регулярные выражения!
> Подайте-ка сюда список заголовков кейсов.
```awk
awk '/^title:/ {print}' portfolio.txt
```

> [!TIP]
> Лучший способ **спокойно** освоить регулярки.\
> Ведь основная с ними проблема — нужны редко и внезапно.

###### Значения полей не так просты, как мы мечтали
> Показывай в строке с названием кейса все поля, кроме первого.
```awk
awk '/^title:/ { $1=""; print $0 } ' portfolio.txt
```
- Интересные ощущения: нас подводит привычка к циклам.

> [!WARNING]
> И тут `awk` не блещет.\
> Зато блещут ==sed== и ==cut==, но не сегодня.

###### Как всегда, суперсила приходит с булевыми значениями
> Проверь, что мы не рехнулись.
```awk
awk '(2 + 2 == 5) { print "Help me!" }' portfolio.txt
```
…ну, это скорее **не**выполнение условия.

###### У вас ещё и встроенные переменные есть?!
> Покажи все названия кейсов и их номера строк.
```awk
awk '/^title:/ { $0 = NR" "$0; print }' portfolio.txt
```

###### Ну а если у нас не строчки?
> Сколько кейсов в портфолио?
```awk
awk -v RS=--- 'BEGIN {sum = 1} { sum = sum + 1 } END { print sum }' portfolio.txt
```
- Можно не инициировать переменную.

###### Грязно, но можно применить даже параметры вызова
> Покажи всё вокруг седьмой строки.
```awk
awk '(NR > ARGV[2] - 3) && (NR < ARGV[2] + 3) { print }' portfolio.txt 7 2>/dev/null
```
 - [?] Как обрабатывать только один файл? Помогут, например, ARGV[1] и FILENAME
 - [!] Лучше, конечно, `-v` использовать 

###### И даже поля можно разделять по шаблону, а не по символам
> Сколько раз использован приём ==R2==?
```awk
awk -v RS=/R2/ '{ SUM += 1 } END { print SUM }' scrambles.csv
```

###### Нашлось место и ассоциативным массивам (словарям)
> Какой язык у нас сколько раз упоминается?
```awk
awk -v FS=, 'NF { LANG[$1]=+1 } END { for (l in LANG) print l, LANG[l] }' lang.csv
```
- Типичный вопрос: «Сумма значений в одной колонке по ключам из другой».


###### Удочка на сладкое — динамическая генерация команд
> Выполняем команды по списку
```awk
awk '{cmd = $1; system(cmd)}' cmd.txt
```
- `$0` разрешит даже параметры для этих команд
- Конечно, **конкретно здесь** `bash` лучше

###### И конечно, встраивание в pipeline
> Сколько там у нас занято места в этой папке?
```awk
ll | awk '{SUM += $5} END {printf("%d\n", SUM/(1024 * 1024))}'
```

> [!TIP]
> Удобно практиковать основы.\
> Переменные, условные ветвления, циклы и даже функции.\
> Проще их практиковать в коротких быстрых программах.\


##### Лайфхаки с булевыми переменными
- `+$1` — «ставит + перед числом», что срабатывает, только если в поле именно число, иначе получаем +0; так проверяем, число ли в поле
- `NF` — число полей равно нулю для пустой строки; так фильтруем пустые строки
- `/./` — строка содержит хотя бы один непробельный символ; так фильтруем пустые строки или строки из одних пробелов
- `a[$0]++` — строка уже встречалась в тексте; аналогично и для полей; `!` для первого вхождения строки
- `awk '$1=$1' FS=, OFS=';' file.txt` — вроде ничего не происходит, но — переформатирование строки; без параметров просто удаляет двойные пробелы
- `$1=$1 || 1` — переформатирование, но сохраняем пустые строки
- `awk '{ print $1 }' FS=, ORS=' ' file.txt; echo` — вывести колонку в строку
> [!WARNING] Это не про запас, это для тренировки
